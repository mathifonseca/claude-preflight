# Contributing to /preflight

Thanks for your interest! Contributions of all sizes are welcome — bug reports, prompt refinements, new check sections, documentation fixes.

## How to contribute

1. **Open an issue first** for any non-trivial change so we can align on scope before you write code.
2. **Fork and branch**: create a feature branch from `main`.
3. **Make your change**: keep it focused. One logical change per PR.
4. **Test locally**: install your fork into `~/.claude/skills/preflight` and exercise the change against a real project.
5. **Open a PR** with a clear description of what changed and why.

## What makes a good contribution

- **New check sections**: propose in an issue first. The bar is "would this be useful to most projects?"
- **Prompt improvements**: share a concrete before/after. The SKILL.md prompts are load-bearing — subtle phrasing changes can shift behavior meaningfully.
- **Bug reports**: include your `.claude/preflight.yaml`, the command you ran, what happened, and what you expected.
- **Documentation**: README fixes, schema examples, and clarifying comments are always welcome.

Be kind. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
